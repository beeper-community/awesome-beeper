# Contribution Guidelines

Thank you for contributing to Awesome Beeper!

## Adding to the List

Please ensure your pull request adheres to the following guidelines:

- **Search existing entries** before adding a new one to avoid duplicates
- **Use the following format:** `[Name](link) - Description`
- **Keep descriptions concise** and end with a period
- **Add new entries alphabetically** within their section
- **Check your spelling and grammar**
- **Remove any trailing whitespace**
- **Ensure the link works** and points to the correct resource

## Categories

### When to add to each category:

- **Official Resources** - Only official Beeper content (website, repos, docs)
- **SDK & API** - Developer tools, libraries, and API documentation
- **Community Tools** - Third-party tools built by the community
- **Bridges** - Bridge-related resources and documentation
- **Tutorials & Guides** - How-to content, tutorials, and guides
- **Community** - Discussion forums, chat rooms, social links

## Quality Standards

We only accept resources that are:

- **Actively maintained** or still relevant
- **Working** and accessible
- **Valuable** to the Beeper community
- **Legal** and respectful of Beeper's terms of service

## Pull Request Process

1. Fork the repository
2. Create a new branch: `git checkout -b add-resource-name`
3. Make your changes
4. Commit: `git commit -m "Add [resource name]"`
5. Push: `git push origin add-resource-name`
6. Create a Pull Request

## Questions?

If you're unsure about anything, open an issue and we'll help!
